# เลขนำโชคเหนือขอบเขต_(Hard)

ก่อนการแข่งขันนัดสำคัญ นักฟุตบอลคนหนึ่งไปขอเบอร์เสื้อนำโชคจากหมอดูชื่อดัง หมอดูไม่ได้บอกเบอร์ตรง ๆ แต่ส่งเลขปริศนาขนาดใหญ่มากมาให้ 2 ค่า โดยเขียนตัวเลขทั้งหมดเป็น **เลขไทย** ตั้งแต่ `๐` ถึง `๙`

เลขปริศนาสองค่านั้นคือ `B` และ `C` ซึ่งตามคำทำนายควรเป็นเลขที่ติดกันพอดี

```text
C = B + 1
```

หมอดูซ่อนเบอร์นักฟุตบอลนำโชค `L` ไว้ในเลข `B` ด้วยสูตร

```text
B = 2^64 + L - 7
```

จำนวนที่หมอดูให้มาอาจมีค่ามากกว่าขอบเขตของชนิดข้อมูลจำนวนเต็มทั่วไปในภาษา C และเลขสองค่าที่ต่างกันเพียง 1 ใกล้ระดับ `2^64` อาจถูกเก็บเป็นค่าเดียวกันถ้าใช้ floating-point เช่น `double`

จงเขียนโปรแกรมเพื่อถอดเบอร์นักฟุตบอลนำโชค `L` ออกมา หากข้อมูลไม่ถูกต้องให้แสดง `INVALID`

## ข้อมูลนำเข้า (Input)

| **ลำดับ** | **รายละเอียด** |
|----------|----------------|
| บรรทัดที่ 1 | สตริง `B` แทนจำนวนเต็มไม่ลบที่เขียนด้วยเลขไทย |
| บรรทัดที่ 2 | สตริง `C` แทนจำนวนเต็มไม่ลบที่เขียนด้วยเลขไทย |

โดยมีเงื่อนไข

```text
1 <= จำนวนหลักของ B, C <= 1000
B และ C ประกอบด้วยเลขไทย ๐ ถึง ๙ เท่านั้น
```

## ข้อมูลส่งออก (Output)

| **ลำดับ** | **รายละเอียด** |
|----------|----------------|
| บรรทัดที่ 1 | แสดงเบอร์นักฟุตบอลนำโชค `L` เป็นเลขอารบิก ถ้า `C = B + 1` และสามารถหา `L` ที่ไม่ติดลบได้ หรือแสดง `INVALID` ถ้าข้อมูลไม่ถูกต้อง |
